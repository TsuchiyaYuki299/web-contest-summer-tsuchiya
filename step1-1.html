Git

① リポジトリの初期化 git init

まず、プロジェクトのフォルダをVSCodeで開きます。
1. 左側のアクティビティバーから「ソース管理」アイコン (枝分かれのようなマーク) をクリックします。
2. 「リポジトリを初期化する」ボタンをクリックすると、そのフォルダがGitリポジトリとして管理されるようになります (git initが実行されます)。

② 変更を記録する git addと git commit

ファイルの変更を記録する手順です。
1. ファイルを変更・保存すると、「ソース管理」ビューの「変更」欄にファイルが表示されます。
2. 記録したいファイルの「+」アイコン (変更をステージ) をクリックして、ステージングエリアに移動させます。これは git add に相当します。
3. 上部のメッセージボックスにコミットメッセージ (例: 最初のコミット) を入力します。
4. 「コミット」ボタンをクリックすると、変更がローカルリポジトリに記録されます (git commit)。

GitHubとの連携

① GitHubへの公開 git push

1. 「ソース管理」ビューで、「リポジトリを公開」ボタンをクリックします。
2. VSCodeがGitHubへのサインインを求めてくるので、ブラウザで認証します。
3. リポジトリ名を入力し、公開範囲 (PublicまたはPrivate) を選択して公開します。
これで、ローカルのコードがGitHub上にアップロード (git push) されます。

② 変更をGitHubにアップロード git push

ローカルでコミットした変更をGitHubに反映させます。
1. VSCodeの左下のステータスバーにある、矢印が円を描いている同期ボタンをクリックします。
2. 確認ダイアログが表示されたら「OK」をクリックすると、ローカルのコミットがGitHubにプッシュされます。

③ GitHubの変更をローカルにダウンロード (プル) git pull

他の人が変更した内容など、GitHub上の最新の状態をローカルに持ってきたい場合も、同様に同期ボタンをクリックします。これにより、リモートの変更がプル ( git pull) されます。

④ 既存のGitHubリポジトリを使う git clone

すでにGitHub上に存在するリポジトリをローカルに持ってきたい場合はクローンします。
1. VSCodeを開き、Ctrl+Shift+P (macOS: Cmd+Shift+P) でコマンドパレットを開きます。
2. Git: Clone と入力して選択します。
3. クローンしたいGitHubリポジトリのURLを貼り付けます。
4. ローカルのどこに保存するかを選択すると、リポジトリのダウンロードが始まります。
